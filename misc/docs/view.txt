Система поддерживает рабуту с несколькими темами и лёгкое переключение между ними.
Для настройки тем в файле настроек в блоке view используются параметры
defaultTheme, currentTheme и dir
diк - каталог в котором хранятся все темы.
defaultTheme, currentTheme - каталоги внутри dir для темы по умолчанию и текущей темы соответственно.
Если система не может найти какой-либо шаблон в текущей теме - она попытается найти его в дефолтной теме

Для создания шаблонов Вы можете использовать любой шаблонизатор на ваш выбор. Для этого нужно создать свой адаптер для шаблонизатора.
Вместе с модулем View постовляются адаптеры для шаблонизаторов Twig, Smarty, так же можно использовать в качестве шаблонов *.php файлы
Каждый модуль может использовыть любой из шаблонизаторов на выбор. Настройка шаблонизатора для темы/модуля осцществляется в файлах engine.cnf.xml

Для унификации вывода используются лэйауты (layout).

Стандартный файл лэйаута находится в корне темы и называется layout.***
Этот лэйаут может быть переопределён для каждого конкретного модуля, контроллера или действия.

Для переопределения лэйаута для всего модуля создайте файл  [ModuleName]/layout.***
Для переопределения лэйаута для контроллера и всех его действий создайте файл  [ModuleName]/Controller/[ControllerName]/layout.***
Для переопределения лэйаута для одного действия создайте файл [ModuleName]/Controller/[ControllerName]/action/[actionName]/layout.***


Отображение шаблонов
Система использует компонентную архитектуру. В качестве компонента можно использовать любой экземпляр любого класса.
Нужно лишь создать для него файл шаблона, который будет храниться в специальном файле, название и путь которого характеризуется пространством имён и цепочкой наследования класса породившего экземпляр (компонент).
Можно сказать, что шаблоны наследуются подобно классам.
